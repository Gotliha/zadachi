# Задачі: Оцінка складності алгоритмів — розв'язки

## Оцінка складності фрагментів коду

### Задача 1: Простий цикл

Цикл виконується рівно `n` разів, а тіло циклу має сталу складність `O(1)`.

**Відповідь:** `O(n)`.

### Задача 2: Вкладені цикли

Зовнішній цикл виконується `n` разів. Для кожної ітерації зовнішнього циклу внутрішній цикл також виконується `n` разів.

Кількість операцій: `n · n = n²`.

**Відповідь:** `O(n²)`.

### Задача 3: Залежний внутрішній цикл

Для кожного `i` внутрішній цикл виконується `i` разів:

```text
0 + 1 + 2 + ... + (n - 1) = n(n - 1) / 2
```

Після відкидання констант і молодших членів залишається квадратичний ріст.

**Відповідь:** `O(n²)`.

### Задача 4: Цикл із поділом

На кожній ітерації `x` ділиться на `3`:

```text
n, n/3, n/9, n/27, ...
```

Потрібно знайти кількість ітерацій `k`, після яких `n / 3^k <= 1`, тобто `3^k >= n`. Звідси `k = log₃ n`.

**Відповідь:** `O(log n)`.

### Задача 5: Послідовні фрагменти

Складності фрагментів:

1. перший цикл: `O(n)`;
2. вкладені цикли: `O(n²)`;
3. цикл на `1000` ітерацій: `O(1)`.

Загальна складність:

```text
O(n) + O(n²) + O(1) = O(n²)
```

**Відповідь:** `O(n²)`.

### Задача 6: Цикл із множенням

На кожній ітерації `x` множиться на `2`:

```text
1, 2, 4, 8, ..., 2^k
```

Цикл завершується, коли `2^k >= n`, тобто `k = log₂ n`.

**Відповідь:** `O(log n)`.

### Задача 7: Хитрий вкладений цикл

Зовнішній цикл має значення `i`:

```text
1, 2, 4, 8, ..., < n
```

Тому зовнішній цикл виконується `O(log n)` разів. Внутрішній цикл для кожної ітерації зовнішнього виконується `n` разів.

Загальна кількість операцій:

```text
n · log n
```

**Відповідь:** `O(n log n)`.

## Доведення за означенням

### Задача 8: Довести `5n + 7 = O(n)`

Потрібно знайти такі `C > 0` і `n₀`, що для всіх `n >= n₀`:

```text
5n + 7 <= Cn
```

Для `n >= 1` маємо `7 <= 7n`, тому:

```text
5n + 7 <= 5n + 7n = 12n
```

Можна взяти:

```text
C = 12, n₀ = 1
```

**Висновок:** `5n + 7 = O(n)`.

### Задача 9: Довести `n² + 3n + 1 = Θ(n²)`

#### Верхня межа: `O(n²)`

Для `n >= 1`:

```text
3n <= 3n²
1 <= n²
```

Тоді:

```text
n² + 3n + 1 <= n² + 3n² + n² = 5n²
```

Можна взяти:

```text
C₂ = 5, n₀ = 1
```

Отже, `n² + 3n + 1 = O(n²)`.

#### Нижня межа: `Ω(n²)`

Для всіх `n >= 1`:

```text
n² + 3n + 1 >= n²
```

Можна взяти:

```text
C₁ = 1, n₀ = 1
```

Отже, `n² + 3n + 1 = Ω(n²)`.

**Висновок:** оскільки функція одночасно є `O(n²)` і `Ω(n²)`, то `n² + 3n + 1 = Θ(n²)`.

### Задача 10: Довести `log n = O(n)`

Потрібно показати, що існують `C` і `n₀`, для яких:

```text
log₂ n <= Cn
```

Для всіх `n >= 1` виконується:

```text
log₂ n <= n
```

Тому можна взяти:

```text
C = 1, n₀ = 1
```

**Висновок:** `log n = O(n)`.

### Задача 11: Чи правда, що `n² = O(n)`?

Ні, це неправда.

Припустимо протилежне: існують `C > 0` і `n₀`, такі що для всіх `n >= n₀`:

```text
n² <= Cn
```

Для `n > 0` поділимо обидві частини на `n`:

```text
n <= C
```

Але `n` може бути як завгодно великим. Наприклад, якщо взяти `n > C`, отримаємо суперечність.

**Висновок:** `n² != O(n)`. Насправді `n² = Ω(n)`.

## Рекурсивна складність

### Задача 12: Рекурсивний факторіал

Кожен виклик робить `O(1)` роботи та викликає функцію для `n - 1`.

Рекурентне співвідношення:

```text
T(n) = T(n - 1) + O(1)
T(0) = O(1)
```

Розкриття:

```text
T(n) = T(n - 1) + O(1)
     = T(n - 2) + 2O(1)
     = ...
     = T(0) + nO(1)
     = O(n)
```

**Відповідь:** `O(n)`.

### Задача 13: Рекурсивна степінь

#### Варіант А: лінійна рекурсія

Кожен виклик зменшує `n` на `1`.

```text
T(n) = T(n - 1) + O(1)
```

**Відповідь:** `O(n)`.

#### Варіант Б: швидке піднесення

Для парного `n` задача зменшується вдвічі:

```text
T(n) = T(n / 2) + O(1)
```

Для непарного `n` спочатку виконується перехід до `n - 1`, після чого число стає парним. Це додає лише сталу кількість роботи перед поділом навпіл.

**Відповідь:** `O(log n)`.

### Задача 14: Дерево рекурсії для `T(n) = 3T(n/3) + O(n)`

На кожному рівні рекурсії:

- рівень `0`: один вузол розміру `n`, робота `O(n)`;
- рівень `1`: три вузли розміру `n/3`, сумарна робота `3 · O(n/3) = O(n)`;
- рівень `2`: дев'ять вузлів розміру `n/9`, сумарна робота `9 · O(n/9) = O(n)`.

На кожному рівні виконується `O(n)` роботи.

Кількість рівнів дорівнює `log₃ n`, бо розмір задачі щоразу ділиться на `3`.

Загальна складність:

```text
O(n) · O(log n) = O(n log n)
```

**Відповідь:** `O(n log n)`.

### Задача 15: Рекурсія з двома гілками

Функція двічі викликає себе для `n / 2`, а потім виконує цикл на `n` ітерацій.

Рекурентне співвідношення:

```text
T(n) = 2T(n / 2) + O(n)
```

Це класичний випадок, аналогічний сортуванню злиттям. На кожному рівні рекурсії сумарна робота `O(n)`, а кількість рівнів `O(log n)`.

**Відповідь:** `O(n log n)`.

## Порівняння росту функцій та границі

### Задача 16: Знайти границю

Потрібно обчислити:

```text
lim x→∞ log x / x²
```

Маємо невизначеність `∞ / ∞`, тому застосуємо правило Лопіталя:

```text
lim x→∞ log x / x² = lim x→∞ (1/x) / (2x) = lim x→∞ 1 / (2x²) = 0
```

**Відповідь:** `0`.

### Задача 17: Знайти границю

Потрібно обчислити:

```text
lim x→∞ x³ / 5ˣ
```

Це невизначеність `∞ / ∞`. Застосуємо правило Лопіталя тричі:

```text
lim x→∞ x³ / 5ˣ
= lim x→∞ 3x² / (5ˣ ln 5)
= lim x→∞ 6x / (5ˣ (ln 5)²)
= lim x→∞ 6 / (5ˣ (ln 5)³)
= 0
```

**Відповідь:** `0`; правило Лопіталя потрібно застосувати **3 рази**.

### Задача 18: Порівняти порядки росту

Функції в порядку зростання від найповільнішої до найшвидшої:

```text
1, log n, √n, n, n log n, n², 2ⁿ, n!
```

### Задача 19: Визначити відношення

#### a) `f(n) = n²`, `g(n) = n³`

Оскільки `n²` росте повільніше за `n³`:

```text
f(n) = O(g(n))
```

Також `f(n) != Ω(g(n))` і `f(n) != Θ(g(n))`.

#### b) `f(n) = 2n + 100`, `g(n) = n`

Лінійна функція з константним доданком має той самий порядок росту, що й `n`.

```text
f(n) = Θ(g(n))
```

Отже, одночасно `f(n) = O(g(n))` і `f(n) = Ω(g(n))`.

#### c) `f(n) = log₂ n`, `g(n) = log₁₀ n`

За формулою переходу між основами:

```text
log₂ n = log₁₀ n / log₁₀ 2
```

Множник `1 / log₁₀ 2` є константою, тому логарифми різних основ відрізняються лише сталим множником.

```text
f(n) = Θ(g(n))
```

Отже, одночасно `f(n) = O(g(n))` і `f(n) = Ω(g(n))`.

## Практичні задачі

### Задача 20: Який алгоритм ефективніший?

Алгоритм A: `100n`.

Алгоритм B: `n²`.

#### a) Найменше `n`, для якого A швидший

Потрібно:

```text
100n < n²
```

Для `n > 0` поділимо на `n`:

```text
100 < n
```

Найменше ціле `n`:

```text
n = 101
```

**Відповідь:** алгоритм A стає швидшим, починаючи з `n = 101`.

#### b) Час для `n = 10⁶`, якщо комп'ютер виконує `10⁹` операцій/с

Алгоритм A:

```text
100n = 100 · 10⁶ = 10⁸ операцій
10⁸ / 10⁹ = 0.1 с
```

Алгоритм B:

```text
n² = (10⁶)² = 10¹² операцій
10¹² / 10⁹ = 1000 с
```

**Відповідь:** A — `0.1 с`, B — `1000 с`, тобто приблизно `16 хв 40 с`.

### Задача 21: Максимальний розмір за 1 секунду

Комп'ютер виконує `10⁸` операцій за секунду. Нехай доступний ліміт — `10⁸` операцій.

#### a) `O(n)`

```text
n <= 10⁸
```

**Відповідь:** `n ≈ 100 000 000`.

#### b) `O(n log n)`

Вважаємо, що `log n` — це `log₂ n`. Потрібно розв'язати:

```text
n log₂ n <= 10⁸
```

Наближено:

```text
n ≈ 4 523 071
```

**Відповідь:** `n ≈ 4.5 · 10⁶`.

#### c) `O(n²)`

```text
n² <= 10⁸
n <= √10⁸ = 10⁴
```

**Відповідь:** `n ≈ 10 000`.

#### d) `O(2ⁿ)`

```text
2ⁿ <= 10⁸
n <= log₂ 10⁸ ≈ 26.58
```

Найбільше ціле `n`:

```text
n = 26
```

**Відповідь:** `n ≈ 26`.
