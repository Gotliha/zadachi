# Задачі: Хеш-функції — розв'язки

## Обчислення хешів

### Задача 1: Хеш цілих чисел

Дано:

```text
h(x) = x mod 11
```

Обчислимо значення для кожного ключа:

```text
15 mod 11 = 4
27 mod 11 = 5
44 mod 11 = 0
11 mod 11 = 0
33 mod 11 = 0
0 mod 11 = 0
-3 mod 11 = 8
```

Для від'ємного числа використовуємо математичний остачевий клас:

```text
-3 ≡ 8 (mod 11), бо -3 + 11 = 8
```

**Відповідь:**

| Ключ | Хеш |
|---:|---:|
| `15` | `4` |
| `27` | `5` |
| `44` | `0` |
| `11` | `0` |
| `33` | `0` |
| `0` | `0` |
| `-3` | `8` |

**Колізії:** ключі `44`, `11`, `33`, `0` мають однаковий хеш `0`.

### Задача 2: Поліноміальний хеш рядка

Дано:

```text
p = 31
m = 100
h(s) = (s0 · p^0 + s1 · p^1 + s2 · p^2) mod m
```

Нумерація літер:

```text
a = 1, b = 2, c = 3, d = 4, ...
```

#### Рядок `"bad"`

Для `"bad"`:

```text
b = 2
a = 1
d = 4
```

Обчислення:

```text
h("bad") = (2 · 31^0 + 1 · 31^1 + 4 · 31^2) mod 100
         = (2 · 1 + 1 · 31 + 4 · 961) mod 100
         = (2 + 31 + 3844) mod 100
         = 3877 mod 100
         = 77
```

#### Рядок `"dab"`

Для `"dab"`:

```text
d = 4
a = 1
b = 2
```

Обчислення:

```text
h("dab") = (4 · 31^0 + 1 · 31^1 + 2 · 31^2) mod 100
         = (4 · 1 + 1 · 31 + 2 · 961) mod 100
         = (4 + 31 + 1922) mod 100
         = 1957 mod 100
         = 57
```

**Відповідь:**

```text
h("bad") = 77
h("dab") = 57
```

Хеші **не збігаються**, отже колізії між цими двома рядками немає.

### Задача 3: Хеш-функція на C++

Потрібно реалізувати:

```text
h(x) = ((a · x + b) mod p) mod m
```

Реалізація:

```cpp
int hash_linear(int x, int a, int b, int p, int m) {
    return ((a * x + b) % p) % m;
}
```

Щоб коректно працювати також з від'ємними `x`, можна нормалізувати остачу:

```cpp
int hash_linear(int x, int a, int b, int p, int m) {
    int value = (a * x + b) % p;
    if (value < 0) {
        value += p;
    }
    return value % m;
}
```

Тест для `a = 3`, `b = 7`, `p = 101`, `m = 10`:

```text
h(x) = ((3x + 7) mod 101) mod 10
```

| `x` | Обчислення | Хеш |
|---:|---|---:|
| `0` | `((3 · 0 + 7) mod 101) mod 10 = 7 mod 10` | `7` |
| `1` | `((3 · 1 + 7) mod 101) mod 10 = 10 mod 10` | `0` |
| `2` | `((3 · 2 + 7) mod 101) mod 10 = 13 mod 10` | `3` |
| `3` | `((3 · 3 + 7) mod 101) mod 10 = 16 mod 10` | `6` |
| `4` | `((3 · 4 + 7) mod 101) mod 10 = 19 mod 10` | `9` |
| `5` | `((3 · 5 + 7) mod 101) mod 10 = 22 mod 10` | `2` |
| `6` | `((3 · 6 + 7) mod 101) mod 10 = 25 mod 10` | `5` |
| `7` | `((3 · 7 + 7) mod 101) mod 10 = 28 mod 10` | `8` |
| `8` | `((3 · 8 + 7) mod 101) mod 10 = 31 mod 10` | `1` |
| `9` | `((3 · 9 + 7) mod 101) mod 10 = 34 mod 10` | `4` |

**Відповідь:** для ключів `0, 1, 2, ..., 9` хеші дорівнюють `7, 0, 3, 6, 9, 2, 5, 8, 1, 4`.

## Колізії та принцип Діріхле

### Задача 4: Знайти колізії

Дано:

```text
h(x) = x mod 5
```

Візьмемо три різні натуральні числа, які мають однакову остачу при діленні на `5`:

```text
1 mod 5 = 1
6 mod 5 = 1
11 mod 5 = 1
```

**Відповідь:** числа `1`, `6`, `11` мають однаковий хеш `1`.

Це завжди можливо, бо натуральних чисел нескінченно багато, а можливих хешів лише `5`: `0`, `1`, `2`, `3`, `4`. За принципом Діріхле, якщо розкласти достатньо багато чисел по 5 комірках, то в деякій комірці опиниться кілька чисел.

### Задача 5: Принцип Діріхле

Є `13` студентів і лише `12` місяців року.

Уявімо, що місяці — це комірки, а студенти — предмети, які потрібно розкласти по цих комірках за місяцем народження.

Якби в кожному місяці народилося не більше одного студента, то студентів було б не більше `12`. Але студентів `13`.

**Висновок:** хоча б двоє студентів народилися в одному місяці.

### Задача 6: Мінімальна кількість колізій

Дано:

```text
m = 8
кількість ключів = 20
```

#### a) Мінімальна кількість ключів у найбільш заповненій комірці

Розподіляємо 20 ключів по 8 комірках якомога рівномірніше:

```text
20 = 8 · 2 + 4
```

Це означає, що в кожну комірку можна покласти по `2` ключі, і ще `4` ключі залишаться. Ці 4 ключі доведеться додати в 4 різні комірки.

Отримаємо розподіл:

```text
3, 3, 3, 3, 2, 2, 2, 2
```

**Відповідь:** у найбільш заповненій комірці буде щонайменше `3` ключі.

#### b) Чи можна розподілити 20 ключів так, щоб у кожній комірці було не більше 3?

Так, можна. Наприклад:

```text
3 + 3 + 3 + 3 + 2 + 2 + 2 + 2 = 20
```

**Відповідь:** так, можна.

## Birthday paradox

### Задача 7: Ймовірність колізії

Дано:

```text
m = 100
k = 5
```

Ймовірність того, що серед 5 випадкових ключів не буде жодної колізії:

```text
P(без колізій) = 100/100 · 99/100 · 98/100 · 97/100 · 96/100
               = 1 · 0.99 · 0.98 · 0.97 · 0.96
               = 0.90345024
```

Ймовірність того, що колізія буде:

```text
P(колізія) = 1 - P(без колізій)
           = 1 - 0.90345024
           = 0.09654976
```

**Відповідь:**

```text
P(без колізій) ≈ 90.35%
P(колізія) ≈ 9.65%
```

### Задача 8: Оцінка кількості елементів

Використовуємо грубу оцінку з умови:

```text
k ≈ sqrt(m)
```

#### a) `m = 1000`

```text
k ≈ sqrt(1000) ≈ 31.62
```

**Відповідь:** приблизно `32` елементи.

#### b) `m = 1 000 000`

```text
k ≈ sqrt(1 000 000) = 1000
```

**Відповідь:** приблизно `1000` елементів.

## Практичні задачі

### Задача 9: Порівняння хеш-функцій

Дано рядки:

```text
"abc", "bca", "cab"
```

Нехай, як і в попередній задачі з поліноміальним хешем:

```text
a = 1, b = 2, c = 3
m = 100
p = 31
```

#### Функція A: сума кодів символів `mod m`

```text
h_A(s) = сума кодів символів mod 100
```

Обчислення:

```text
h_A("abc") = (1 + 2 + 3) mod 100 = 6
h_A("bca") = (2 + 3 + 1) mod 100 = 6
h_A("cab") = (3 + 1 + 2) mod 100 = 6
```

**Висновок:** усі три рядки мають однаковий хеш `6`, тобто утворюють колізію.

#### Функція B: поліноміальний хеш

```text
h_B(s) = (s0 · 31^0 + s1 · 31^1 + s2 · 31^2) mod 100
```

Оскільки:

```text
31^0 = 1
31^1 = 31
31^2 = 961
```

Обчислення для `"abc"`:

```text
h_B("abc") = (1 · 1 + 2 · 31 + 3 · 961) mod 100
           = (1 + 62 + 2883) mod 100
           = 2946 mod 100
           = 46
```

Обчислення для `"bca"`:

```text
h_B("bca") = (2 · 1 + 3 · 31 + 1 · 961) mod 100
           = (2 + 93 + 961) mod 100
           = 1056 mod 100
           = 56
```

Обчислення для `"cab"`:

```text
h_B("cab") = (3 · 1 + 1 · 31 + 2 · 961) mod 100
           = (3 + 31 + 1922) mod 100
           = 1956 mod 100
           = 56
```

**Відповідь:**

| Рядок | Функція A | Функція B |
|---|---:|---:|
| `"abc"` | `6` | `46` |
| `"bca"` | `6` | `56` |
| `"cab"` | `6` | `56` |

**Яка функція краща?** Функція B краща, бо враховує порядок символів. Функція A не враховує порядок, тому будь-які перестановки тих самих символів мають однаковий хеш. У цьому прикладі функція B все ще має колізію для `"bca"` і `"cab"`, бо модуль `m = 100` малий, але вона розрізняє більше випадків, ніж проста сума.

### Задача 10: Хеш-функція для точок

Потрібно, щоб точки `(1, 2)` і `(2, 1)` мали різні хеші. Тому не можна використовувати лише `x + y`, бо:

```text
1 + 2 = 2 + 1
```

Використаємо різні коефіцієнти для `x` та `y`:

```cpp
struct point {
    int x;
    int y;
};

int hash_point(const point& p, int m) {
    int value = (31 * p.x + 17 * p.y) % m;
    if (value < 0) {
        value += m;
    }
    return value;
}
```

Перевірка для `(1, 2)` і `(2, 1)`:

```text
hash_point({1, 2}, m) = (31 · 1 + 17 · 2) mod m = 65 mod m
hash_point({2, 1}, m) = (31 · 2 + 17 · 1) mod m = 79 mod m
```

Наприклад, якщо `m = 100`, то:

```text
hash_point({1, 2}, 100) = 65
hash_point({2, 1}, 100) = 79
```

**Відповідь:** така функція повертає значення в межах `{0, ..., m - 1}` і для `m = 100` розрізняє точки `(1, 2)` та `(2, 1)`.
